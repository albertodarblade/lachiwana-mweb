# Implementation Plan: PWA Offline Configuration

**Branch**: `006-pwa-offline-config` | **Date**: 2026-05-02 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/006-pwa-offline-config/spec.md`

## Summary

Configure the Lachiwana mobile web app as a Progressive Web App with offline read-only support. A hand-written service worker delivers cache-first loading for the app shell; TanStack Query's `persistQueryClient` persists the query cache to localStorage so user data (notebooks, notes, attachment metadata) survives page reloads while offline. Two Framework7 UI components surface offline status and available updates to the user.

**Language/Version**: JavaScript/JSX (React 19, ES modules)

## Implementation Notes

### Service Worker (`public/sw.js`)

- Cache name: `lachiwana-static-v1` — bump version suffix to invalidate on deploy.
- `install` event: pre-cache the app shell (paths: `/`, `/offline.html`, and all bundled JS/CSS files). Use `self.skipWaiting()` in `install` only if auto-update is desired; otherwise omit and let `UpdateBanner` drive it via `SKIP_WAITING` message.
- `activate` event: delete all cache buckets whose name does not match `lachiwana-static-v1`. Call `self.clients.claim()`.
- `fetch` event:
  - If request is cross-origin → pass through (no caching).
  - If request is for a static asset (JS, CSS, image, font) → cache-first: serve from cache; if miss, fetch and cache.
  - If request is a navigation (`request.mode === 'navigate'`) → network-first: try network; on failure serve cached `/` or `/offline.html`.
- `message` event: if `event.data.type === 'SKIP_WAITING'` → call `self.skipWaiting()`.

### TanStack Query Persistence (`src/queryClient.js`)

- Install: `pnpm add @tanstack/react-query-persist-client @tanstack/query-sync-storage-persister`
- Wrap `QueryClient` with `persistQueryClient({ queryClient, persister, maxAge: 24 * 60 * 60 * 1000 })`.
- Use `createSyncStoragePersister({ storage: window.localStorage, key: 'LACHIWANA_QUERY_CACHE' })`.
- Clear persisted cache on logout: call `localStorage.removeItem('LACHIWANA_QUERY_CACHE')` inside `clearSession()` in `authStore.js`.
- `staleTime` on `QueryClient` should remain at `10_000` ms; `gcTime` should be extended to `24h` to prevent cache entries from being garbage-collected before the persister writes them.

### Service Worker Registration (`src/main.jsx`)

```
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js').then(registration => {
      // detect update: listen for updatefound
      registration.addEventListener('updatefound', () => {
        const newWorker = registration.installing
        newWorker.addEventListener('statechange', () => {
          if (newWorker.state === 'installed' && navigator.serviceWorker.controller) {
            // signal UpdateBanner
          }
        })
      })
    })
  })
}
```

Pass `waitingWorker` state down from `main.jsx` to `App.jsx` via a top-level React state or a simple module-level event emitter following the `networkStore` pattern.

### Offline/Update Banners (`src/components/`)

- Use Framework7 `Block` with inline styles for the fixed positioning (Framework7 does not expose a built-in fixed notification bar; build from `Block` + `Link` primitives per Constitution IV).
- `OfflineBanner` reads `isOnline` via `useNetwork()`.
- `UpdateBanner` receives `waitingWorker` as a prop from `App.jsx`.
- Both are rendered inside `App.jsx` as siblings to `<F7App>`.

### Write-Action Blocking

No new code required for FR-005 in this phase: existing mutation hooks (`useCreateNote`, `useUpdateNote`, etc.) will fail gracefully because API requests fail when offline. 

However, for a better UX, each mutation hook that fires on user action should check `getNetworkStatus()` before calling the API and throw/reject with a user-visible message if `isOnline === false`. This surfaces the offline constraint before the network request fails. This is an optional enhancement documented here but not required for the MVP acceptance criteria.
